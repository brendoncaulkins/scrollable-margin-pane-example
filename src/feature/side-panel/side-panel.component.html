There are three components here:

  • App
  • Side-Panel
  • Main-Panel

App - contains the grey background, and fills the view, vertically by using height: 100vh;

We are also using Flex layout, throughout. display: flex; puts the elements into a row, which sits the Side and Main panels side-by-side;

Side Panel - nothing special here, just uses a white background to cover up the overall grey one.

Main Panel - Has all the magic, thanks to a pair of nested containers, the __wrapper and the __content within.

The __wrapper inherits the full-screen grey background, and is also going to manage our vertical scroll.

First, we make use Flex again to establish a column arrangement. While there will be only one item in our column (the __content), creating a flex column means that we can use flex to force the child to always fill all available vertical space.

Then, we enable a scroll area with scroll: auto;, and scroll-x: hidden;. This enables a vertical scroll bar when needed, and hides the horizontal scroll, always. (When/if there is a need for a horizontal scroll, the components rendering that content should implement their own horizontal scroll, like with a wide table, for example).

The __content inner div establishes the "card" style look, with the white background and rounded corners.

The flex: 1; asks this inner container to fill teh space if it's parent in the primary direction, which is vertically for a column layout. This is why the card does not shrink when there is empty space below the paragraphs.

The main trick after that is the margin: 16px 16px; which creates the transparent space around the __content for the __wrapper to show through, with the grey background.
